Data Flow Reference

How Valet Master processes data from login to logout — online vs offline.

Login
1
ScreenApp launches — splash screen
POST/auth/validate-token  +  POST/device/register
parallel calls — check token validity + register device
Token valid → skip login
Navigate to open cash / dashboard
Token invalid → login screen
POST/auth/login
email + password + device_id → token + is_open_cash
Open Cash
2
POST/cash-sessions/start
opening_balance → res: shift UUID
DriftSave shift row — shifts table
server shift id stored, syncStatus: synced
GET/branches/:id   GET/rates   GET/vehicle-types   GET/settings
background hydration — branch config + rates + overnight cutoff
DriftSave to branch_config + rates tables
used for offline fee computation at checkout
Check-In
3
POST/transactions
contact_number + valet_type → res: server UUID (server_ticket_id)
DriftSave draft — tickets table
server_ticket_id saved — required for all subsequent sync
PATCH/transactions/:id
vehicle + driver_in + parking + belongings + condition + signature
DriftUpdate ticket row — status: active, syncStatus: synced
PrintThermal ticket — attendant copy + customer claim stub + key tag
Check-Out
4
QR scan
DriftLookup by server_ticket_id
local first
GET/transactions/:id
remote fallback if not found locally
Manual TKT-XXXX
DriftLookup by ticket_number
local first
GET/tickets/by-number/:n
remote fallback if not found locally
On-deviceFee computed from cached Drift rates — no server call
flat_rate + succeeding_hours + overnight_fee if applicable
PATCH/transactions/:id
driver_out + condition_checkout + status: active
POST/transactions/:id/pay
method: cash + amount_paid → res: change
DriftUpdate ticket — status: completed, syncStatus: synced
fee + driverOut + damageMarkers persisted
PrintReceipt — change displayed, car released
Close Cash + Logout
5
POST/cash-sessions/close
shift_id + actual_cash → res: status: closed
POST/auth/logout
fire-and-forget — session already cleared locally before this call
Login
1
ScreenApp launches — splash screen detects offline
InternetReachability.hasInternet() returns false
DriftOffline login — read from offline_accounts table
no server call — cached credentials used
DriftRates + config — read from last synced cache
rates, branch_config tables used as-is
Open Cash
2
DriftWrite shift row — shifts table
opening_balance saved, syncStatus: pending
Queueshifts/create → sync_queue
POST /cash-sessions/start will fire when online
Check-In
3
DriftWrite draft — tickets table
local TKT-XXXX generated — no server_ticket_id yet
Queuetickets/create → sync_queue
POST /transactions will fire first to get server UUID
DriftUpdate ticket — full check-in data
vehicle + driver_in + belongings + condition + signature
Queuetickets/update (check-in) → sync_queue
PATCH payload queued — fires after POST gets server UUID
PrintThermal ticket — works offline
Check-Out
4
DriftLookup ticket — local only
no remote fallback — if not on device, cannot proceed until online
On-deviceFee computed from cached Drift rates
flat_rate + succeeding_hours + overnight_fee if applicable
DriftUpdate ticket — checkout condition + payment
driver_out + condition_checkout + fee + status: completed
Queuetickets/update (checkout) + pay → sync_queue
PATCH condition then POST pay — queued in strict order
PrintReceipt — works offline, car released
Sync — when connectivity returns
5
SyncCubitDetects connectivity — flushes sync_queue in order
POST/transactions → get server_ticket_id
must fire first — UUID required for all subsequent calls
DriftSave server_ticket_id → ticket row
subsequent queue items resolve :id from this UUID
PATCH/transactions/:id — check-in payload
vehicle + driver_in + belongings + condition
PATCH/transactions/:id — checkout condition
driver_out + condition_checkout + status
POST/transactions/:id/pay
amount_paid → change (only if fee > 0)
DriftAll rows marked syncStatus: synced
Close Cash + Logout
6
Queueshifts/close → sync_queue (if still offline)
POST /cash-sessions/close fires when connectivity returns
ScreenSession cleared locally — logout server call skipped if offline
